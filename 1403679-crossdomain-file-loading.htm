crossdomain文件加载

主页 / 嵌入 / crossdomain文件加载

本指南介绍了在HTML5浏览器/设备使用时，跨域文件加载(安全性)限制使用Adobe Flash插件和JavaScript。

该法不允许的域的访问内容是一个安全性的措施,防止用户基本上leeching您的文件,并使用它们在其他地方。 此功能是一个白名单,在这所有的域都天生就被阻止,除非指定。 走马观花式的一个基本的过程如下所示:

  1. 一个JW播放器上托管domaina希望访问一个文件,是VTT主持domainb上。
  2. domainb认为这一请求,并确认domaina应具有访问权限。
  3. 如果domaina是允许的访问,则该文件。

每个版本、FLASH和html5,有一种不同的方法的允许/拒绝跨域访问权。 现在让我们来探讨不同之处。

crossdomainFLASH

Adobe Flash播放器包含一个 crossdomain安全机制,类似于JavaScript的 跨站点脚本执行 限制。 它会影响瑞草区盘浦洞JW发布者使用的FLASH播放器强大的浏览器和设备,拒绝加载以下文件在另一个域:

  • RSS订阅源
  • XML外观
  • VTT、斯普斯卡广播电视台和dfxp文件
  • M3U8清单和TS片段
  • SMIL清单

crossdomain XML

安全限制crossdomain可解除托管一个 crossdomain.xml 文件的服务器上包含该文件。

例如,如果请求domaina.com从一个文件domainb.com,您将需要配置crossdomain.xml domainb.com授予访问权限。  此crossdomain文件必须放置在根的域或子域。 例如: 

http://www.myserver.com/crossdomain.xml
http://videos.myserver.com/crossdomain.xml

在FLASH播放机将尝试加载XML文件,SWF文件或原始数据来自任何域以外的一个托管您的播放器,它会检查远程站点中的存在着这种一个crossdomain.xml文件。 如果FLASH认为,如果配置允许外部访问的数据,则数据加载。 如果没有,则操作将是不允许的。

允许所有示例

这里是一个示例的一个crossdomain.xml,允许访问此域的SWF文件中的数据在任何站点: 

<?xml版本="1"?">"
<!DOCTYPE跨域的策略系统“http://www.adobe.com/xml/dtds/cross-domain-policy.dtd">
<跨域策略>
<允许的访问从domain="*"/>
</跨域策略>

请注意,此示例设置您的服务器。 任何SWF文件可以装入任何数据从您的站点,这可能会导致安全问题。

限制访问示例

下面是另一个例子crossdomain.xml,这次允许Flash文件访问从仅有的数域: 

<?xml版本="1"?">"
<!DOCTYPE跨域的策略系统“http://www.adobe.com/xml/dtds/cross-domain-policy.dtd">
<跨域策略>
<允许的访问从domain="*.domain1.com"/>
<允许访问的域="www.domain2.com"/>
</跨域策略>

请注意,使用通配符符号:任何子域的域1可以将数据装载,而域2是仅限于www子域。

crossdomain策略文件甚至可以进一步finegrain访问,例如,某些端口或HTTP报头。 一个详细的概述,请参阅 Adobe的crossdomain文档。

 

在JavaScript crossdomain

在JavaScript中,一个 跨站点脚本执行 机制类似 于FLASH 存在。 它会影响出版者使用瑞草区盘浦洞JW播放器对HTML5强大的浏览器和设备,拒绝加载以下文件从另一个域:

  • RSS订阅源
  • XML外观
  • VTT、斯普斯卡广播电视台和dfxp文件

一般来说,这些文件加载将失败如果没有crossdomain访问。 大多数浏览器会显示一个错误的调试控制台。

cross-origin资源共享

crossdomain访问可以是已启用Javascript中类似的机制,在FLASH中。 而不是一个crossdomain.xml文件中,crossdomain访问权限已启用每个文件,通过一个附加HTTP响应头( CoR 标题)。 这里的什么标题如下所示: 

访问控制-allow-origin:*

请注意,此示例设置您的文件。 任何底稿从任何站点可以加载该文件和为所欲为。

限制访问示例

下面是另一个例子COR头,这次允许JavaScript文件只能从一些域: 

访问控制-allow-origin:*.domain1.com www.domain2.com

请注意,使用通配符符号:任何子域的域1可以将数据装载,而域2是仅限于www子域。

选项来限制协议和端口可以被添加。 请参阅enabled-cors.org站点 以了解更多信息。 该站点列出了如何使COR标题等各种流行的Web服务器、框架和服务器端语言。

当使用我们的云计算托管播放机和托管外部文件在您自己的站点(标题、播放列表等),您将需要给我们CDN crossdomain访问。 这可确保外部文件可以正常访问和加载我们的播放器。 “crossdomain.xml文件在一个个人域名应该有以下行添加到确保: 

允许的访问从.jwpcdn.com" domain="*

同样,该域的COR标题应进行调整,以允许该访问。
