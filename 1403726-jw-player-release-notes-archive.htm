Google IMA修补程序

对下列项目进行了固定在6/2015:

对于Google最近IMA SDK 更改,一个补丁,用于我们Google IMA广告插件被推到2015年6月1日。 因此,JW播放器6.12或更高版本 必须用来为HTML5与我们广告Google IMA插件。

除了上述变化,以下功能和改进已添加:

  • 改善AD计划,允许使用<播放列表>反倾销规则,VMAP日程Google IMA。
  • 添加支持创意的非线性在HTML5模式对于非iPhone设备。
  • 改进的视频prerolls时间,防止内容之前播放广告。
  • playad()API调用将在HTML5模式下使用Google时IMA。

JW播放器6.11

发行提高了功能、性能和UX在几个不同的方法,具备的功能包括:

  • 支持HLS协议版本6功能
  • 一个新的模型的事件发送至Google分析与内置插件
  • MP4视频转换从一个Flash播放器一体化通过我们chromecast

HLS更改

  • 支持6HLS协议草案版本13功能:
    • 多个音频曲目样式使用#EXT-x-媒体类型等于音频
    • 生活中的字幕HTTP流使用webvtt#EXT-x-媒体类型等于字幕
  • Flash播放器会检测到超时元数据格式在ID3在嵌入式的MPEG2传输流和泡泡在onmeta()API事件
  • 当#EXT-x-程序的日期-时间标记,onmeta()将火警时播放的适用部分。
  • JW播放器将自动选择一个音频曲目根据系统语言
  • 改进的有效载荷608标题
  • 能力的缓冲区内寻求无需rebuffer
  • 能够重新寻求在媒体是缓冲
  • DVR滑动控件工具栏显示在“可用长度为2分钟(而不是1分钟。

HLS v5不支持的功能

  • 示例-AES解密
  • iFrame缩略图曲目
  • byterange请求
  • 任何超时,元数据是一个映像。

chromecast

  • 铸件控制现在JW播放器时可用FLASH中呈现为播放MP4视频。
  • “领衔主演”图标不显示在源代码是不接收器上播放应用程序。
  • 转换图标已经更新的设计合规性。

一般更新

  • IE9现在仅支持在FLASH、HTML5支持已被删除。
  • 网络代理连接类型可以设置为rtmps连接
  • 海报映像将无法加载时自动启动:true,提高播放器启动速度。
  • WebKit的playsinline被添加为一个属性的标记的视频播放器使用,JW当呈现在IOS设备。

错误修复

  • 固定setuperror射时在某些情况下设置播放器在FLASH中。
  • 全屏按钮固定在互联网资源管理器。
  • 修正了一个问题,即当vpaid失败将被打破,其他大量广告。
  • 当一个广告暂停在Android镀铬,单击它将不再打开点击链接目标,但继续播放。
  • 固定的音量滑块广告回放在FLASH中。
  • 固定标记变量替换项目的持续时间是不更换辊中或开机自检。
  • 固定使用.play()在一个特收看了在FLASH中。
  • 修正了一个问题,自动启动提供商正在被打破。
  • 固定一个crossdomain vpaid加载问题广告。

JW播放器6.10

此功能释放包含品牌特定的功能新JW播放器广告和企业版的JW播放器。

chromecast支持

  • 在ADS控制栏内容不再阻塞在广告播放。 它消失在广告开始播放。
  • 将AD标记现在更容易,内置支持。 现在,您可以通过在多个广告一个广告块以及playad()API回退到多个广告标记。
  • 支持更新googima IMA API*(HTML5支持在之前的版本)

固定

  • 固定一个错误设置YouTube在HTML5模式。
  • VMAP应支持vastaddata以及vastdata。
  • 修正了一个问题,广告创意vpaid正在更多空间定义。

JW播放器6.9

JW播放器6.9有很多新功能和更新,是所有版本中均可-键盘快捷方式和YouTube HTML5支持。 Google通用分析是现在支持同时保持支持旧格式。 企业版有已建即用功能一体化chromecast使用Google。 在进行这些更新您的很多收藏昆虫已经挤压。

新

  • 支持通用分析Google在内置GA插件。 维护支持较旧的数据模型。
  • JW播放器现在将支持HTML5无边浏览YouTube播放器! 这允许HTML5视频上传到YouTube视频播放播放器在JW在移动和台式机!
  • 键盘快捷方式! JW播放器现在可以控制键盘控制简单的Flash和html5。 选项卡放入播放机和使用空间,以开始播放,用箭头键寻求和调节音量,并能够快速进入/退出全屏。
  • 第二个准确地寻求在HLS损益表。 在寻求的HLS流中,而不是在开始的一个片段可以寻求直接到第二个。
  • Android上镀铬全屏视频播放自定义的JW播放器面板。

更新

  • onmeta返回ID3元数据的当前片段播放AAC HLS音频流。
  • JW播放器现在支持AES解密的高质量高清HLS流。 播放器会对其进行解密和播放高清内容加密速度更快6倍,比以前有可能流高质量保护的内容。
  • JW播放器从碧蓝、HLS损益确认而无需进行任何其他配置。
  • JW播放器有时会无法加载如果不能达到CDN,外部插件。 现在的播放器将启动即使某些插件无法加载。
  • 最新全屏使用遵守已过时浏览器API调用。
  • 更新了Twitter徽标在共享插件,符合品牌的标准。

广告

  • onadplay()-API调用来确定广告的播放时,返回adtag,位置的广告如果它是在一个adpod和状态下,它是移动,将播放或暂停。
  • onadpause()API事件时的广告是“暂停”。
  • 时间滑块提示不再显示非线性广告。
  • onadimpression adtitle属性包含从巨大的。
  • 单击一个vpaid特不再强制暂停的视频播放。 暂停视频回放点击的vpaid创意。
  • playad()googimaFLASH

测试版支持chromecast

  • 企业和广告客户可以领衔主演MP4内容到一个JW播放器chromecast接收器应用程序提供一个无缝的集成。
  • 支持线性大量广告。
  • 支持基本的自定义操作和品牌的接收器的应用程序。

错误

  • 608标题成为固定的后同步采取行动。
  • 固定后出现iPad控件时退出播放多个播放视频(播放列表、广告)。
  • 固定不变的广告vpaid考虑整个播放器包括可视播放列表,并不仅仅是播放器窗口中。
  • 寻求问题解决方法当您单击时间滑块的广告标记。
  • 某些大量呼叫被超速缓存的播放列表。 这种超速缓存不再发生。
  • 修正了一个问题,即当标题是VTT溢出而不是行打破。
  • 固定式徽标时显示没有自动启动:是设置。
  • 固定一个对齐问题全屏视频播放在Safari。
  • 固定播放中的对齐与HLS准确适合在全屏显示。
  • 固定广告播放时,HTML5的巨大mediafile反应没有。
  • 固定标注多个源文件的格式不正确。
  • vpaid1.1标签的问题,因为播放的一个缺少getvpaid方法。 播放器将播放这些类型的vpaid标签。

版本6.8

2-2014:6.8JW播放器带来了重大更新,广告的巨大的支持,VMAP规格3。 下一步,该播放器包括完全重新设计Pro外观和支持符合FCC标准标题造型优美,令人耳目一新。

新增功能

  • 3大量合规性,其中包括:
    • 可跳过广告
    • 特抗倒伏能力强
    • 进度和跳过跟踪器
    • 标准错误代码
    • 包装程序解析更新
  • VMAP1.0的兼容性:能够解析特安排一个V形图标签。
  • 最新设计和想象Pro面板(8)。
  • FCC合规性的字幕支持进一步自定义的标题样式选项。

更新和增强功能

  • 中间辊AD标记现在显示在timeslider。
  • 仅音频MP3的创意发挥在JW播放器。

  • 固定播放问题与某些vpaid AD标记adparameters使用。
  • 固定定位问题与某些DFP非线性叠加广告执行接管操作。
  • 固定timeslider缩略图,没有格式化时正常使用多个精灵在FLASH中。

  • YouTube网站上的IOS iframe后退7现在是正确的格式。
